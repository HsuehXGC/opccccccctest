# OPC · 虚拟人力中枢

> 让一个人管理多个 Claude CLI 机器人，完成企业任务。

OPC 是给小业主用的「虚拟人力」操作台。你录入企业需求，系统把它拆成任务，
再把任务派给一支由 **Claude CLI 机器人**组成的虚拟团队去执行——你只需要在
看板上盯进度、做复核。

## 三大模块

| 模块 | 作用 |
| --- | --- |
| **需求管理** | 以「产品」为中心的工作台：需求以文档形式（Markdown 正文 + 关联文档 + 派生任务）管理，一站式，和产品文档双向打通 |
| **产品文档** | 按产品生命周期分层的 Wiki：产品为根、10 类文档（核心+可选）、类型化关系（派生/实现/验证/决策）、文档蓝图与缺口、混合版本历史 |
| **任务看板** | 工作单元流水线：文档/执行任务分型、交付物闭环、执行简报、轻量依赖、按产品筛选、任务详情抽屉 |
| **虚拟人力** | 部署/管理机器人（每个是一个独立 Claude CLI 会话），查看在岗状态与实时工位 |

核心链路：**产品 → 需求（文档形式）→ 产出文档 / 拆解任务 → 部署机器人执行 → 看板追踪**

### 账户与多租户结构

`账户组（Root + 成员）→ 项目（多个，共享）→ 产品 → 需求/文档/任务`

- **账户体系**：每个用户一个 Root 账户，可绑定多个成员账户（编辑/只读/管理员），构成一个账户组。侧边栏底部账户 chip 进入「团队与账户」
- **项目**：账户组共享多个项目（适合代理/多客户）。侧边栏顶部切换项目；切换后全站内容（产品/需求/文档/任务）随之隔离
- **虚拟人力账户级共享**：机器人属于账户组、跨项目共用，不随项目切换——契合下方本地算力模型
- **本地算力**：账户绑定本地电脑，每台电脑跑一个个 `claude` / `codex` 执行器，即虚拟人力背后真实算力。「团队与账户」页可「绑定电脑」（一行安装 + 一次性 token）、查看在线机器与执行器
- 模型见 [`frontend/src/types.ts`](frontend/src/types.ts) 的 `Account` / `Project` / `Machine` / `Executor`；`Product` 挂 `projectId`，内容按当前项目作用域过滤

### 内网机器接入 · 常驻 agent 架构

用户机器多在内网（无公网 IP、进不去），所以让机器**主动出站**建立常驻连接，指令顺着这条连接回传：

- **传输**：机器上跑常驻 agent，出站建 `wss:443` 长连接到云端网关；只需出站，无需公网 IP / 开入站端口（SSH 作为可选通用通道后续叠加）
- **协议** [`backend/src/agentProtocol.ts`](backend/src/agentProtocol.ts)：`enroll` / `heartbeat` / `job:dispatch` / `job:chunk|done|error`。一次性 enroll token（账户作用域）换长期 agentToken
- **网关** [`backend/src/agentGateway.ts`](backend/src/agentGateway.ts)：`AgentGateway` 管理所有 agent 连接——注册、心跳、按执行器派单、回传中继（传输无关，真实 WS 服务器在 M2 挂到 `/agent`）
- **派单闭环**：`task.brief` → 绑定的执行器（某台电脑上的 claude/codex）→ `gateway.dispatch()` → 机器上跑 → 输出经隧道流回看板
- **绑定电脑流程**：`团队与账户 → 绑定电脑` 给出 Mac/Windows/Ubuntu 的一行安装命令（含 token）→ agent 出站接入 → 自动探测执行器

### 任务看板 · 工作单元流水线

任务是「需求/文档」与「虚拟员工执行」之间的桥。每个任务是一个完整的工作单元：

- **文档 / 执行 分型**：文档任务产出/修订某篇产品文档；执行任务产出工作成果（代码/分析/文案）
- **交付物闭环**：文档任务移到「已完成」时，自动为目标文档生成一个新版本（note 标注「由任务 X 交付」）——任务系统与文档系统闭环
- **执行简报**：每个任务带一段 `brief`（给员工的指令 + 验收标准），即将来 `claude -p <brief>` 的输入；任务详情抽屉里可编辑。这是任务对接机器人执行层的接缝
- **轻量依赖 DAG**：任务间可声明「A 阻塞 B」，看板卡片标「被 N 个任务阻塞」，抽屉里可增删依赖
- **产品筛选 + 任务详情抽屉**：看板按产品过滤；点卡片打开抽屉（状态流 / 简报 / 交付物 / 依赖 / 负责员工 / 执行日志）
- 数据模型见 [`frontend/src/types.ts`](frontend/src/types.ts) 的 `Task`（`kind` / `brief` / `targetDocSlug` / `output` / `dependsOn`）

### 需求管理 · 产品工作台

- **产品线切换**：左栏切换产品，主区为该产品的一站式工作台（需求 / 文档蓝图缺口 / 派生任务概览）
- **需求即文档**：每条需求是一张可展开的文档卡——Markdown 正文（支持 `[[slug]]` 关联产品文档）、优先级、可改状态、编辑
- **双向打通**：需求正文里的 `[[链接]]` 与「关联文档」卡片可一键跳到产品文档中心并定位到对应文档（跨模块深链，导航状态提到 store）；关联文档由 `doc.requirementId` 自动聚合，派生任务由 `task.requirementId` 聚合
- **蓝图缺口感知**：工作台顶部显示核心文档覆盖（如 `核心 1/4`），缺口直接给「去文档中心补齐」入口

### 产品文档 Wiki · 文档架构

按产品生命周期分层的**文档蓝图**。五层架构：

1. **组织根 = 产品**：`产品 → 产品版本 → 文档`。产品是一级实体、文档树的根命名空间；小业主可同时管理多个产品（Wiki 顶部切换）。
2. **分类体系（生命周期 → 类型）**：五阶段 10 种类型，分**核心**（PRD/架构/接口/测试，每产品默认应有）与**可选**（愿景BRD/用户故事/数据模型/视觉/ADR/发布）。
   - ① 定义 · 愿景/BRD ② 规划 · PRD/用户故事 ③ 设计 · 架构/接口/数据模型/视觉/ADR ④ 验证 · 测试 ⑤ 发布 · 发布说明
3. **类型化关系**：文档间用有向关系互链——`派生自 / 实现 / 验证 / 决策 / 引用`，正反双向展示、可增删。工作流据此判断上下游、自动建链。正文内 `[[slug]]` 保留为阅读用的通用引用（「正文引用」）。
4. **混合版本模型**：每篇文档独立迭代（v1/v2/v3 快照 + 时间线），并标注所服务的**产品版本**；状态流 草稿→评审→定稿→归档，可预览/回滚。
5. **文档蓝图（工作流接缝）** ⭐：一个产品的蓝图 = 该有哪些文档 + 彼此怎么连。现存文档 = 已实现实例，**蓝图 − 现状 = 缺口**。Wiki 左侧「文档蓝图」实时显示核心覆盖（如 `核心 4/4`）与缺口的「起草」入口。将来工作流接入时：读蓝图 → 找缺口 → 派对应角色员工按模板起草 → 自动建立类型化链接 → 走状态流。

数据模型见 [`frontend/src/types.ts`](frontend/src/types.ts)（`Product` / `WikiDoc` / `DocRelation` / `DocVersion`）。M2 可平滑落到 `docs/` 真实 `.md` 文件（frontmatter 存产品/版本/类型/关系）。

## 技术栈

- **前台**：Vite + React + TypeScript + Tailwind v4 + Zustand
- **后端**：Node + Express，`AgentGateway` 管理内网机器的常驻 agent 连接，按执行器派单
- monorepo（npm workspaces）：`frontend/` + `backend/`

## 快速开始

```bash
npm install            # 安装全部 workspace 依赖

npm run dev            # 启动前台 (http://localhost:5173)
npm run dev:api        # 另开一个终端，启动后端 (http://localhost:8787)
```

> 里程碑 1：前台完整可交互，由浏览器内 **mock 数据** + 模拟执行引擎驱动，
> 无需后端即可体验全部流程。侧边栏的「执行引擎」可暂停/启动模拟。

## 从 mock 切到真实执行（M2 落点）

1. `backend/src/agentGateway.ts` — `AgentGateway` 已就位（enroll / heartbeat / dispatch / 回传中继）
2. `backend/src/index.ts` — 在 `/agent` 挂 `WebSocketServer`，把每条 agent 连接包成 `AgentConnection` 交给 `gateway`；派单走 `POST /api/tasks/:id/dispatch`，`GET /api/stream` 走 SSE
3. 实现跨平台 agent（Go/Rust 单二进制 + OS 服务），按 `agentProtocol.ts` 收发；探测本机 claude/codex 账户上报为执行器
4. 前台把 bot 绑定到执行器，`assignTask` / 派单改为调用后端，用 SSE 流替换模拟日志
