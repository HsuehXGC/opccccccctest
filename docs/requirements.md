# OPC · 需求文档（PRD）

## 1. 产品定位

OPC（虚拟人力中枢）是一个面向**小业主 / 代理团队**的操作台：让一个人管理多个
Claude / Codex CLI 机器人，把企业需求拆解为文档与任务，交给一支"虚拟团队"执行，
自己只需在看板上盯进度、做复核。

一句话：**一个人，一支虚拟团队。**

## 2. 目标用户

- 独立开发者 / 小业主：一个人要交付一个或多个产品
- 代理 / 工作室：同时服务多个客户（多项目）
- 共同点：本地有若干台电脑（Mac / Windows / Ubuntu），上面装着 claude / codex，
  希望把这些算力编排起来干活

## 3. 术语表

| 术语 | 说明 |
| --- | --- |
| 账户（Account） | 用户身份。分 Root（组织主账户）与成员（Member） |
| 账户组 / 组织（org） | 一个 Root + 其绑定的成员账户 |
| 项目（Project） | 账户组共享的顶层工作区，含多个产品 |
| 产品（Product） | 项目下的一级内容实体，文档树的根 |
| 需求（Requirement） | 文档形式的业务需求，可拆解为任务 |
| 产品文档（WikiDoc） | 覆盖生命周期的文档（PRD/架构/接口…），带类型化关系与版本 |
| 任务（Task） | 交给虚拟员工执行的工作单元（文档任务 / 执行任务） |
| 虚拟人力 / 机器人（Bot） | 一个 Claude/Codex CLI 会话的抽象，账户组内共享 |
| 机器（Machine） | 账户绑定的本地电脑 |
| 执行器（Executor） | 机器上一个 claude / codex 账户，机器人背后的真实算力 |

## 4. 核心链路

```
账户 → 项目 → 产品 → 需求（文档形式）
                          ├─ 产出产品文档（PRD/架构/接口/测试…）
                          └─ 拆解为任务 → 派虚拟员工执行 → 看板追踪 → 交付物回写
```

虚拟员工的执行最终落到**本地算力**：机器人绑定到某台电脑上的某个 claude/codex 执行器，
"派机器人执行任务" = 把任务简报发到那台电脑上跑。

## 5. 功能需求

### 5.1 账户与项目（多租户）
- 每个用户一个 Root 账户，可绑定多个成员账户（编辑 / 只读 / 管理员）
- 账户组共享多个项目；切换项目时全站内容（产品/需求/文档/任务）随之隔离
- 虚拟人力属于账户组，跨项目共享

### 5.2 需求管理 · 产品工作台
- 以产品为中心的工作台：选产品 → 该产品的需求、文档蓝图缺口、派生任务一站式
- 需求以**文档形式**管理：Markdown 正文（支持 `[[slug]]` 关联产品文档）、优先级、状态
- 需求与产品文档双向打通（正文互链一键跳转）

### 5.3 产品文档 Wiki
- 按生命周期分层的 10 类文档，分核心（PRD/架构/接口/测试）与可选（愿景/故事/数据模型/ADR/发布）
- **类型化关系**：派生自 / 实现 / 验证 / 决策 / 引用；正反双向展示
- **混合版本模型**：每篇独立迭代（v1/v2/v3 快照 + 时间线），并标注所服务的产品版本
- 生命周期状态：草稿 → 评审 → 定稿 → 归档
- **文档蓝图**：实时显示核心文档覆盖与缺口，缺口给"起草"入口

### 5.4 任务看板 · 工作单元流水线
- 四列拖拽流转：待分配 → 进行中 → 待复核 → 已完成
- **文档 / 执行**分型；文档任务完成自动为目标文档生成新版本（交付物闭环）
- **执行简报**：每个任务带一段给员工的指令 + 验收标准（= `claude -p` 的输入）
- **轻量依赖**：任务间可声明"A 阻塞 B"，看板标记被阻塞
- 按产品筛选；任务详情抽屉（状态流 / 简报 / 交付物 / 依赖 / 负责员工 / 执行日志）

### 5.5 虚拟人力
- 部署/管理机器人（角色、模型、技能、状态）
- 实时工位与在岗状态

### 5.6 本地算力 · 绑定电脑
- 账户绑定本地电脑：一行安装命令（含一次性 token），agent 出站接入
- 自动探测机器上的 claude / codex 账户，登记为执行器
- 查看在线机器与执行器状态

## 6. 非功能需求

- **跨平台**：agent 覆盖 Mac / Windows / Ubuntu，单二进制 + OS 服务
- **内网友好**：用户机器多在内网、无公网 IP；接入只需**出站** `wss:443`，不开入站端口
- **多租户隔离**：内容按 org / project 作用域隔离
- **安全**：一次性短时效 enroll token；凭证进 KMS 不明文长期存；最小权限；可一键吊销并断连
- **可观测**：执行日志实时流式回传

## 7. 里程碑

- **M1 — 完整前台 + mock（已完成）**：多租户、四大模块、Wiki、任务流水线、绑定电脑流程
- **M2 — 真实执行闭环**：真实 agent 接入，看板派单经 agent 在机器上跑 `claude -p`，SSE 回传
- **M3 — 持久化 + 鉴权**：数据库、真实登录、KMS、需求→任务自动拆解（规划机器人）
